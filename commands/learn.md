# /learn - 再利用可能なパターンの抽出

現在のセッションを分析し、skill として保存する価値のあるパターンを抽出します。

## トリガー

非自明な問題を解決したタイミングで、セッション中いつでも `/learn` を実行します。

## 抽出する対象

以下を探します：

1. **エラー解決パターン**
   - どのようなエラーが発生したか？
   - 根本原因は何か？
   - 何が修正に繋がったか？
   - 類似のエラーに再利用できるか？

2. **デバッグ手法**
   - 自明でないデバッグ手順
   - 有効だったツールの組み合わせ
   - 診断パターン

3. **回避策**
   - ライブラリの癖
   - API の制限
   - バージョン固有の修正

4. **プロジェクト固有のパターン**
   - 発見されたコードベースの慣習
   - 行われたアーキテクチャ上の決定
   - 統合パターン

## 出力形式

`~/.claude/skills/learned/[パターン名].md` に skill ファイルを作成します：

```markdown
# [説明的なパターン名]

**抽出日:** [日付]
**コンテキスト:** [適用されるシーンの簡単な説明]

## 問題
[これが解決する問題 - 具体的に]

## 解決策
[パターン・手法・回避策]

## 例
[該当する場合はコード例]

## 使いどころ
[トリガー条件 - この skill を有効化すべきタイミング]
```

## プロセス

1. セッションを見直して抽出可能なパターンを探す
2. 最も価値があり再利用性の高い知見を特定する
3. skill ファイルの草稿を作成する
4. 保存前にユーザーに確認を求める
5. `~/.claude/skills/learned/` に保存する

## 注意事項

- 些細な修正（タイポ、単純な構文エラー）は抽出しない
- 一時的な問題（特定の API 障害など）は抽出しない
- 将来のセッションで時間を節約できるパターンに焦点を当てる
- skill は 1 つのパターンに絞る
