# Code Review

未コミットの変更に対する包括的なセキュリティ・品質レビューを行います：

1. 変更ファイルの取得: git diff --name-only HEAD

2. 各変更ファイルについて以下を確認：

**セキュリティ上の問題（CRITICAL）：**
- ハードコードされた認証情報、API キー、トークン
- SQL インジェクションの脆弱性
- XSS の脆弱性
- 入力バリデーションの欠如
- 安全でない依存関係
- パストラバーサルのリスク

**コード品質（HIGH）：**
- 50 行を超える関数
- 800 行を超えるファイル
- ネストの深さが 4 段階を超えるもの
- エラーハンドリングの欠如
- console.log 文
- TODO/FIXME コメント
- パブリック API に対する JSDoc の欠如

**ベストプラクティス（MEDIUM）：**
- ミューテーションパターン（代わりにイミュータブルを使用）
- コード・コメント内の絵文字の使用
- 新規コードに対するテストの欠如
- アクセシビリティの問題（a11y）

3. 以下の形式でレポートを生成：
   - 重要度: CRITICAL、HIGH、MEDIUM、LOW
   - ファイルの場所と行番号
   - 問題の説明
   - 修正案

4. CRITICAL または HIGH の問題が見つかった場合はコミットをブロック

セキュリティの脆弱性があるコードは絶対に承認しないこと！
