# レビューモード

作業モード: PR レビュー・コード分析
フォーカス: 品質・セキュリティ・保守性

## チェックリスト

### セキュリティ（最優先）
- [ ] ハードコードされたシークレット・API キーがないか
- [ ] SQL インジェクションの可能性がないか
- [ ] XSS の可能性がないか
- [ ] 認証・認可の漏れがないか
- [ ] 入力値のバリデーションがあるか

### コード品質
- [ ] 関数が 50 行以内か
- [ ] ファイルが 800 行以内か
- [ ] ネストが 4 レベル以内か
- [ ] エラーハンドリングがあるか
- [ ] テストが書かれているか（80%+）

### 保守性
- [ ] 命名が明確か
- [ ] 重複コードがないか
- [ ] 適切なコメント（WHY を説明）があるか

## 出力形式

ファイル別・重要度順（CRITICAL → HIGH → MEDIUM → LOW）で報告する。
80% 以上の確信度がある指摘のみ報告する。

## 使うツール

Read, Grep, Glob, Bash
