# 開発モード

作業モード: アクティブ開発
フォーカス: 実装・コーディング・機能構築

## 基本方針

- コードを書くことを優先、説明は後でする
- 動く解決 > 完璧な解決
- テストを書いてから実装する（TDD）
- 変更はアトミックに（小さくシンプルに保つ）
- 新しいコードを書く前に既存コードを検索する（search-first）

## 優先順位

1. 動かす → 2. 正しくする → 3. きれいにする

## 使うツール

Edit, Write, Bash, Grep, Glob

## チェックリスト（実装完了時）

- [ ] テストが通っている
- [ ] console.log / print() が残っていない
- [ ] ハードコードされたシークレットがない
- [ ] ビルドが成功している
